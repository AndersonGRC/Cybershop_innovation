Crear tenant nuevo

Esto provisiona un cliente nuevo end-to-end: crea su base de datos, aplica el schema base, registra en el control plane y emite la primera API key.

¿Qué hace este botón exactamente?
  1. Valida que el slug sea único.
  2. Computa el nombre de BD (cyber_tNNN) y hace CREATE DATABASE.
  3. Aplica el schema base (productos, usuarios, pedidos, etc.).
  4. Siembra los datos del cliente: usuario administrador con el email del dueño + contraseña temporal, nombre de la empresa, colores de marca, secciones del sitio y datos de contacto.
  5. Registra el cliente en el control plane con el plan elegido y activa los módulos que incluye ese plan.
  6. Emite la primera API key + client code.
  7. Aprovisiona la instancia (puerto, dominio slug.cybershopcol.com y SSL).

Tarda 2-5 segundos. Al terminar te mostramos la API key y la contraseña del administrador UNA SOLA VEZ — copialas y guardalas en lugar seguro.
