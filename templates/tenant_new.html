Crear tenant nuevo

Esto provisiona un cliente nuevo end-to-end: crea su base de datos, aplica el schema base, registra en el control plane y emite la primera API key.

¿Qué hace este botón exactamente?
  1. Valida que el slug sea único.
  2. Computa el nombre de BD (cyber_tNNN).
  3. CREATE DATABASE cyber_tNNN en Postgres.
  4. Aplica el schema base (productos, usuarios, pedidos, etc.).
  5. Inserta filas en tenants y tenant_databases.
  6. Emite la primera API key + client code para el cliente.

Tarda 2-5 segundos. Al terminar te mostramos la API key UNA SOLA VEZ — copiala y guardala en lugar seguro.
